# RAG по PDF-документам университета

Локальный RAG-проект на Python, который индексирует PDF-документы, сохраняет их векторные представления в Qdrant и отвечает на вопросы по найденному контексту через LLM в Groq.

Проект ориентирован на работу с внутренними документами университета: правила приема, приказы, регламенты и другие PDF-файлы из папки `data/`.

В репозитории намеренно хранится только `data/university.pdf` как открытый пример. Остальные PDF-документы считаются локальными рабочими материалами, не должны попадать в git и игнорируются через `.gitignore`.

## Что делает проект

- Читает все PDF-файлы из папки `data/`
- Извлекает обычный текст и содержимое таблиц
- Делит документы на чанки для поиска по смыслу
- Строит эмбеддинги через `sentence-transformers`
- Сохраняет чанки и метаданные в Qdrant
- На вопрос пользователя ищет наиболее релевантные фрагменты
- Передает найденный контекст в Groq и формирует финальный ответ

## Как устроен пайплайн

```text
PDF-файлы из data/
    ->
извлечение текста и таблиц
    ->
разбиение на чанки
    ->
эмбеддинги BAAI/bge-m3
    ->
Qdrant collection: university_docs2
    ->
поиск top-5 релевантных фрагментов
    ->
Groq / Llama 3.3 70B
    ->
готовый ответ пользователю
```

## Стек

- Python
- `PyMuPDF (fitz)` для чтения PDF и таблиц
- `langchain-text-splitters` для чанкования текста
- `sentence-transformers` с моделью `BAAI/bge-m3`
- `Qdrant` для векторного хранилища
- `Groq` для генерации ответа на основе найденного контекста
- `python-dotenv` для загрузки переменных окружения

## Структура проекта

```text
RAG/
|-- data/
|   |-- university.pdf
|   `-- другие локальные PDF-документы
|-- scripts/
|   |-- ingest.py   # индексация документов в Qdrant
|   `-- rag.py      # поиск контекста и генерация ответа
|-- backend/        # пока пустая директория под будущую серверную часть
|-- .env.example
`-- README.md
```

## Основные сценарии

### 1. Индексация базы знаний

Скрипт `scripts/ingest.py` проходит по всем PDF из `data/`, извлекает текст, разбивает его на чанки и загружает в коллекцию `university_docs2` в Qdrant.

Особенности текущей реализации:

- размер чанка: `600`
- overlap: `120`
- размер вектора в коллекции: `1024`
- distance metric: `COSINE`
- каждому чанку добавляется `payload` с текстом и именем исходного файла

### 2. Ответ на вопрос

Скрипт `scripts/rag.py` строит эмбеддинг вопроса, ищет похожие чанки в Qdrant и отправляет собранный контекст в модель `llama-3.3-70b-versatile` через Groq.

По умолчанию:

- берется `top-5` наиболее близких фрагментов
- ответ формируется только на основе найденного контекста
- в файле есть тестовый вызов `ask_question(...)`

## Быстрый старт

### 1. Подготовь окружение

```powershell
python -m venv .venv
.venv\Scripts\activate
```

### 2. Установи зависимости

```powershell
pip install python-dotenv groq qdrant-client sentence-transformers pymupdf langchain-text-splitters
```

Если `sentence-transformers` попросит дополнительные зависимости для PyTorch, установи их стандартным способом через `pip`.

### 3. Запусти Qdrant

Самый простой вариант через Docker:

```powershell
docker run -p 6333:6333 -p 6334:6334 qdrant/qdrant
```

По текущему коду Qdrant ожидается по адресу `http://localhost:6333`.

### 4. Заполни `.env`

Создай файл `.env` на основе `.env.example`.

Минимально для текущего кода нужен:

```env
GROQ_API_KEY=your_groq_api_key
```

Замечание: `GEMINI_API_KEY` в `.env.example` остался от прошлой версии проекта и в текущих скриптах не используется.

### 5. Положи PDF-файлы в `data/`

Скрипт индексации автоматически возьмет все файлы `*.pdf` из этой папки.

Важно:

- `data/university.pdf` можно хранить в репозитории как открытый пример
- остальные PDF в `data/` считаются приватными рабочими документами и не должны публиковаться
- для этого в `.gitignore` настроено исключение: git отслеживает только `university.pdf`

### 6. Выполни индексацию

Запускать лучше из корня репозитория:

```powershell
python scripts/ingest.py
```

После этого чанки документов будут загружены в коллекцию `university_docs2`.

### 7. Запусти RAG-поиск

```powershell
python scripts/rag.py
```

Скрипт выполнит тестовый запрос, зашитый внизу файла, и выведет ответ в консоль.

## Пример использования в коде

```python
from scripts.rag import ask_question

answer = ask_question("Какие есть правила приема?")
print(answer)
```

## Что важно знать перед использованием

### Проект в текущем виде

- Это не готовый веб-сервис, а рабочая база RAG-логики на скриптах
- В репозитории сейчас нет FastAPI-приложения и фронтенда, хотя старый README это упоминал
- Папка `backend/` пока пустая и, вероятно, оставлена под следующий этап разработки

### Ограничения текущей реализации

- Повторный запуск `ingest.py` будет добавлять новые чанки с новыми `uuid`, а не переиспользовать старые записи
- `ingest.py` импортирует Groq, хотя в процессе индексации он фактически не используется
- В коде есть текстовые фрагменты с проблемами кодировки, их стоит привести к UTF-8
- Параметры подключения к Qdrant и имя коллекции пока зашиты прямо в коде
- Нет отдельного файла зависимостей вроде `requirements.txt` или `pyproject.toml`
